## Summary

Build a pytest-driven API regression suite that uses httpx clients to verify account creation, listing/filtering, balance adjustments, and validation/error handling under JWT security, supported by deterministic fixtures and documented execution guidance.

## Technical Context

**Language/Version**: Python 3.12  
**Primary Dependencies**: FastAPI (existing API surface), SQLAlchemy (models), httpx (tests), pytest (runner), python-accounting helpers.  
**Storage**: SQLite (development defaults; tests use in-memory or sqlite file nodes when necessary).  
**Testing**: pytest with httpx AsyncClient fixtures, leveraging existing pytest config and tox/poetry pipelines.  
**Target Platform**: Linux-based development servers/CI agents where uvicorn and pytest run.  
**Project Type**: Web-service API.  
**Performance Goals**: Test suite completes in under five minutes against the local API and surfaces deterministic outputs for every scenario.  
**Constraints**: JWT authentication must remain enabled during CI runs to cover security guards; fixtures must clean up so repeated executions see a blank slate.  
**Scale/Scope**: Focused on account/transaction endpoints (creation/listing/adjustment/balance) exposed by `src/sdd_cash_manager/api`; does not cover UI or external integrations beyond documented APIs for now.
